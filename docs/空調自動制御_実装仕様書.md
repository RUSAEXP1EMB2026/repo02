# 空調自動制御 実装仕様書

**ドキュメント作成日**: 2026-05-13  
**対象ブランチ**: `origin/develop`  
**ステータス**: 進行中（Phase 1 基礎実装済み、拡張機能実装予定）

---

## 概要

本仕様書は、Google Apps Script + Nature Remo 3 を用いた **在宅時自動空調制御システム** の実装要件を定義します。

### スコープ
- **実装完了**: 基本的なセンサ値取得・温度判定・クーラー制御・ログ記録
- **拡張予定**: 除湿モード対応・電気代アラート・外出中の室温監視（Phase 2）

---

## 機能要件

### 1. 温度監視と自動制御

#### 1.1 実装状況

| 項目 | ステータス | 詳細 |
|---|---|---|
| 5分ごとのセンサ取得 | ✅ 実装済み | `runEvery5Minutes()` で実装 |
| 不快指数（DI）計算 | ✅ 実装済み | `calcDiscomfortIndex()` で計算 |
| 温度判定ロジック | ✅ 実装済み | `evaluateAndControl()` で判定 |
| クーラー制御（冷房） | ✅ 実装済み | `controlAircon()` で実行 |
| 稼働状況ログ記録 | ✅ 実装済み | `appendSensorLog()` で記録 |
| 高温アラート | ✅ 実装済み | `sendHighTempAlertNotification()` で通知 |

#### 1.2 制御ロジック

```javascript
// 温度帯判定（Settings シート B2:B3 から取得）
- tempMin = 22℃（下限）
- tempMax = 23℃（上限）

// 制御フロー
if (currentTemp > tempMax) {
  → クーラー稼働（冷房モード）
} else if (currentTemp < tempMin && isAcRunning()) {
  → クーラー停止
} else {
  → 状態維持
}
```

#### 1.3 高温アラート機能

| 項目 | 実装内容 |
|---|---|
| 判定値 | Settings シート B11 から取得（デフォルト: 35℃） |
| 設定キー | `highTempAlertThreshold` |
| 判定条件 | `sensorData.temp > highTempAlertThreshold` |
| 通知機能 | `sendHighTempAlertNotification()` |
| 記録対象 | SensorLog シートに記録 |

### 2. 定時通知機能

#### 2.1 実装状況

| 項目 | ステータス | 詳細 |
|---|---|---|
| 時間帯フィルタ | ✅ 実装済み | `shouldSendScheduledReport_()` で判定 |
| 通知間隔制御 | ✅ 実装済み | `notifyInterval` で制御 |
| スケジュール通知 | ✅ 実装済み | `sendScheduledReport()` で送信 |

#### 2.2 設定パラメータ

| パラメータ | 設定行 | デフォルト | 説明 |
|---|---|---|---|
| `notifyStart` | B5 | 7 | 通知開始時刻（時） |
| `notifyEnd` | B6 | 23 | 通知終了時刻（時） |
| `notifyInterval` | B7 | 60 | 通知間隔（分） |

**注意**: 仕様書では 30分/1時間/2時間の選択設定予定ですが、現状は固定値を分単位で制御します。

### 3. センサデータ記録

#### 3.1 SensorLog シートのスキーマ

| 列 | 項目 | 型 | 説明 |
|---|---|---|---|
| A | タイムスタンプ | DateTime | 測定日時 |
| B | 気温 | Number | 室温（℃） |
| C | 湿度 | Number | 相対湿度（%） |
| D | 照度 | Number | 照度（lux） |
| E | 不快指数 | Number | DI値 |
| F | 空調状態 | String | "稼働"/"停止" |

#### 3.2 OperationLog シートのスキーマ

| 列 | 項目 | 型 | 説明 |
|---|---|---|---|
| A | タイムスタンプ | DateTime | 操作日時 |
| B | 操作内容 | String | 操作の説明 |
| C | 気温 | Number | 当時の室温 |
| D | 湿度 | Number | 当時の湿度 |
| E | 不快指数 | Number | 当時のDI値 |
| F | 結果 | String | "成功"/"失敗" |
| G | エラー詳細 | String | エラー内容（失敗時） |

### 4. Discord 通知機能

#### 4.1 通知タイプ一覧

| 通知タイプ | 関数 | 条件 | 優先度 |
|---|---|---|---|
| クーラー制御通知 | `sendAirconNotification()` | クーラー稼働・停止時 | 高 |
| 定時レポート | `sendScheduledReport()` | 時間帯内・指定間隔 | 中 |
| 高温アラート | `sendHighTempAlertNotification()` | 35℃超過 | 高 |
| エラー通知 | `sendErrorNotification()` | 例外発生時 | 高 |

#### 4.2 通知の内容

```
【クーラー稼働通知】
時刻: 14:30
室温: 25.5℃
湿度: 60%
不快指数: 75
→ クーラーを冷房モードで稼働しました

【定時レポート】
時刻: 18:00
室温: 23.2℃ / 設定値: 22-23℃
湿度: 55%
不快指数: 72
クーラー状態: 停止
```

## 拡張機能（Phase 2以降）

### 1. 除湿（ドライ）モード対応

```javascript
// 実装予定
if (humidity >= humidityThreshold && currentTemp < tempMax) {
  controlAircon(remoToken, deviceName, "dry");  // 除湿モード
}
```

| 項目 | 仕様 |
|---|---|
| 判定条件 | 湿度 ≥ 70% かつ 室温 < 23℃ |
| 制御方法 | Nature Remo API の mode パラメータに "dry" 指定 |
| 通知 | クーラー稼働時と同様に Discord 通知 |

### 2. 外出中の室温監視

```javascript
// GPS外出判定時、5分ごとの高温チェック
if (isUserOutdoor()) {
  if (currentTemp > highTempAlertThreshold) {
    sendHighTempAlertNotification(discordUrl, temp, humidity);
  }
}
```

## 実装ガイドライン

### 1. コード品質

- **言語**: Google Apps Script（ES5相当）
- **関数命名**: キャメルケース（`runEvery5Minutes`）
- **エラーハンドリング**: 全API呼び出しに try-catch
- **ログ出力**: `Logger.log()` で実行履歴を記録

### 2. テスト方法

#### 2.1 単体テスト
```javascript
// Apps Script エディタで実行可能
function testTemperatureControl() {
  var mockSensorData = {
    temp: 25,
    humidity: 60,
    illuminance: 500
  };
  var result = evaluateAndControl(mockSettings, mockSensorData, "停止");
  Logger.log("制御判定: " + JSON.stringify(result));
}
```

#### 2.2 統合テスト
1. Settings シートに設定値を入力
2. Nature Remo トークン・Discord URL を設定
3. `runEvery5Minutes()` を手動実行
4. SensorLog / OperationLog を確認

## トラブルシューティング

### よくあるエラーと対応

| エラー | 原因 | 対応 |
|---|---|---|
| "Sheet not found: Settings" | Settings シートが削除された | シートを再作成 |
| "API token invalid" | Nature Remo トークン期限切れ | B8 を新しいトークンに更新 |
| "Discord URL not found" | Discord Webhook URL 未設定 | B9 に URL を入力 |
| "controlAircon failed" | 機器がオフラインまたは無応答 | Nature Remo アプリで確認 |
