# 設計書

## 設計内容の概要

- 予めユーザーが以下の設定値をスプレッドシートの所定のセルに入力できる欄を設ける。
  - 自宅の緯度・経度・ジオフェンス半径・GPS取得間隔
  - 監視対象機器ごとの有効・無効フラグ
  - 目標温度帯（最低温度・最高温度）※デフォルト22〜23℃
  - Discord Webhook URL

- **在宅/外出の判定ロジック**
  スマートフォン側（IFTTTまたはiOS Shortcuts等）からGPS座標をGASのWebhookに2分ごとに送信する。GAS側でハーバーサイン式により自宅座標との距離を算出し、ジオフェンス半径との比較を行う。ジオフェンス逸脱・復帰いずれも**2回連続**で確認された場合にのみステータスを切り替える（誤検知防止）。

  **距離計算式（ハーバーサイン）**
  d = 2r × arcsin(√(sin²((lat2-lat1)/2) + cos(lat1)×cos(lat2)×sin²((lon2-lon1)/2)))
  （r = 地球半径 6371 km）

---

### 外出時：消し忘れ通知機能

- 外出ステータスへの切り替えと同時に、Nature Remo 3 APIを通じて監視対象として有効化されている全機器の稼働状況（電源オン/オフ、設定値）を取得する。

- 電源オンの機器が1台以上存在した場合、Discord Webhook APIへ以下の内容を含むメッセージをPOSTする。

  **通知メッセージの内容**
  - オン状態の機器名の一覧
  - 各機器の最終オン操作からの経過時間
  - クーラーの場合は現在の設定温度
  - スプレッドシートの操作ダッシュボードへのURL

- ユーザーがダッシュボードから機器のオフ操作を行った場合、Nature Remo 3 APIへ電源オフ命令を送信する。操作日時・機器名・操作結果をスプレッドシートに記録する。

---

### 在宅時：最適温度維持機能

- 在宅ステータス中、5分ごとに Nature Remo 3 のセンサから温度・湿度・照度を取得してスプレッドシートに記録する。不快指数は以下の式で算出する。

  **不快指数 = 0.81 × 温度[℃] + 0.01 × 湿度[%] × (0.99 × 温度[℃] − 14.3) + 46.3**

- 取得した室温とユーザー設定の目標温度帯を比較し、以下のロジックでクーラーを制御する。

  | 条件 | 動作 |
  |------|------|
  | 室温 > 目標温度帯の上限 かつ クーラーがオフ | クーラーを最低設定温度で稼働 |
  | 室温 < 目標温度帯の下限 かつ クーラーがオン | クーラーを停止 |
  | 稼働中に室温が目標帯内に収まっている場合 | 設定変更なし |

- クーラーの稼働・停止時は Discord Webhook APIへ以下の内容を含むメッセージをPOSTする。
  - 操作内容（稼働開始 / 停止）
  - 操作時の室温・湿度・不快指数
  - クーラーの設定温度

- 毎時0分に、スプレッドシートの最新行の温度・湿度・不快指数・クーラー稼働状況をDiscordへ定時報告する。

- 外出ステータスに切り替わった時点で温度管理ループを停止し、在宅に戻った時点で再開する。

## システム処理の流れ

```
[ユーザー] → スプレッドシートに各種設定を入力

━━━ 外出時 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[スマートフォン（IFTTT等）：2分ごと]
  ↓ GPS座標をGAS Webhookに送信
[GAS] → ハーバーサイン距離を計算 → スプレッドシートに記録

[距離 > ジオフェンス半径 が2回連続]
  → 外出判定
  ↓ 監視対象機器の稼働状況を取得
[Nature Remo 3 API] → 機器ごとの電源状態・設定値を返す

[オン状態の機器が1台以上]
  → Discord Webhook へ消し忘め通知をPOST

[ユーザーがダッシュボードからオフ操作]
  ↓
[Nature Remo 3 API] → 電源オフ命令を送信
[スプレッドシート] ← 操作ログを記録

━━━ 在宅時 ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[スケジューラ：5分ごと]
  ↓ 室温・湿度・照度を取得
[Nature Remo 3 API] → センサ値を返す
[スプレッドシート] ← 温度・湿度・不快指数を記録

[室温 > 目標上限 かつ クーラーOFF]
  → Nature Remo 3 API経由でクーラー稼働
  → Discord Webhook へ稼働通知をPOST

[室温 < 目標下限 かつ クーラーON]
  → Nature Remo 3 API経由でクーラー停止
  → Discord Webhook へ停止通知をPOST

[毎時0分]
  → 最新センサ値・稼働状況を Discord Webhook へPOST（定時報告）
```

## 必要なモジュール（.gs ファイル）

- **GPS取得・ジオフェンス判定プログラム**
  スマートフォンからのGPS座標を受け取り、距離計算・在宅/外出ステータス管理を行う。

- **機器状態取得プログラム**
  Nature Remo 3 APIを通じて監視対象機器の電源状態・設定値を取得する。

- **温度管理プログラム**
  在宅時に室温と目標温度帯を比較し、クーラーの稼働・停止を制御する。

- **リモート操作プログラム**
  ユーザーの操作指示を受け取り、Nature Remo 3 APIへ命令を送信する。

- **Discord通知プログラム**
  消し忘め通知・クーラー操作通知・定時報告の文面を生成し、Discord Webhook URLへPOSTする。

- **スプレッドシート管理プログラム**
  ユーザー設定値の読み込み、センサログ・位置情報ログ・操作ログの書き込みを管理する。

## 補足：GPS座標の取得手段

Google Apps Script はスマートフォンのGPSを直接取得できないため、以下いずれかの方法でスマートフォン側からGASのWebhookへ座標を送信する。

| 方法 | 概要 | 難易度 |
|------|------|--------|
| IFTTT（iOS/Android） | 位置情報トリガー → GAS Webhook URLへPOST | 低（設定のみ） |
| iOS Shortcuts（オートメーション） | 到着/出発トリガー → GAS URLへHTTPリクエスト | 低〜中 |
| 専用アプリ（React Native等） | 常時バックグラウンドGPS → GAS APIへ定期送信 | 高（開発が必要） |

プロトタイプ段階ではIFTTTまたはiOS Shortcutsの利用を推奨する。
