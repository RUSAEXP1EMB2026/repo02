# 環境構築ガイド：VSCode + clasp による GAS ローカル開発

## 概要

このドキュメントでは、Google Apps Script (GAS) をローカルの VSCode で開発・管理するための環境構築手順を説明します。  
clasp（Google 公式ツール）を使うことで、GAS のスクリプトをローカルで編集し、GitHub でバージョン管理しながらチーム開発が行えます。

---

## 前提条件

- Git の基本操作（clone / commit / push / pull）を理解していること
- Google アカウントを持っていること
- VSCode がインストール済みであること

---

## ディレクトリ構成（推奨）

```
groupX/
├── README.md
├── .gitignore
├── GASscripts/
│   ├── .claspignore
│   ├── .clasp.json        ← gitignore 対象（絶対にコミットしない）
│   ├── appsscript.json
│   ├── sheet.js
│   ├── remo.js
│   ├── sensordata.js
│   └── discord.js
└── docs/
    ├── 要求仕様書.pdf
    └── 設計書.pdf
```

---

## STEP 1：GAS API を有効化する

1. 以下のリンクを **開発で使用する Google アカウント** でログインした状態で開く  
   👉 https://script.google.com/home/usersettings
2. **「Google Apps Script API」をオンにする**

---

## STEP 2：Node.js をインストールする

1. 以下から LTS 版インストーラを取得してインストール  
   👉 https://nodejs.org/en
2. インストール後、ターミナルで動作確認

```bash
node -v
npm -v
```

---

## STEP 3：Windows のみ — PowerShell 実行ポリシーの変更

> macOS / Linux の場合はこの手順をスキップしてください。

PowerShell を **管理者として実行** し、以下を順に実行する。

```powershell
# 現在のポリシーを確認
Get-ExecutionPolicy

# RemoteSigned に変更
Set-ExecutionPolicy -Scope CurrentUser
# → 入力を求められたら「RemoteSigned」と入力して Enter

# 変更を確認
Get-ExecutionPolicy
# → "RemoteSigned" と表示されれば OK
```

---

## STEP 4：clasp をインストールしてログインする

```bash
# グローバルインストール
npm install -g @google/clasp

# Google アカウントでログイン（ブラウザが開く）
clasp login
```

ブラウザで以下の 2 つの権限を許可する：
- Google Apps Script のデプロイの作成と更新
- Google Apps Script のプロジェクトの作成と更新

---

## STEP 5：GAS プロジェクトをクローンする

1. GAS を開き、URL から `scriptId` を取得する

```
https://script.google.com/home/projects/【ここが scriptId】/edit
```

2. `GASscripts/` ディレクトリを作成し、その中でクローンを実行する

```bash
mkdir GASscripts
cd GASscripts
clasp clone 【scriptId】
```

> クローンすると `.clasp.json`（アクセストークン含む）が生成されます。  
> **絶対に GitHub にコミットしないでください。** → STEP 6 で設定します。

---

## STEP 6：.gitignore と .claspignore を設定する

### .gitignore（リポジトリルートに作成）

```
# clasp の認証情報（絶対にコミットしない）
.clasp.json
```

### .claspignore（GASscripts/ 内に作成）

```
# すべてのファイルを除外した上で、必要なものだけ許可する
**/**

# 許可するファイル
!appsscript.json
!**/*.html
!**/*.js
```

---

## STEP 7：VSCode で開発する

VSCode で `GASscripts/` フォルダを開き、`.js` ファイルを編集する。

> **ファイルの拡張子について**  
> ローカルで新規ファイルを作成する場合は `.js` に統一してください。  
> clasp が push/pull 時に `.gs` と `.js` を自動変換します。  
> `.gs` と `.js` が混在すると重複ファイルが発生するため注意してください。

---

## STEP 8：GAS との同期

### ローカルの変更を GAS に反映する（push）

```bash
cd GASscripts
clasp push
```

### GAS の変更をローカルに取得する（pull）

```bash
clasp pull
```

> チーム内で GAS エディタを直接編集しない運用であれば、`pull` を使う機会はほとんどありません。

---

## STEP 9：GitHub でチーム共有する

「Git の使い方」の資料を参照し、ある程度実装が進んだ段階でコミット・プッシュして共有する。

```bash
# ルートに戻る
cd ..

# 変更を確認
git status

# ステージングとコミット
git add .
git commit -m "feat: ◯◯機能を追加"

# プッシュ
git push origin main
```

---

## 主要コマンド早見表

| コマンド | 説明 |
|---|---|
| `clasp login` | Google アカウントでログイン |
| `clasp clone [scriptId]` | GAS プロジェクトをローカルにクローン |
| `clasp push` | ローカルの変更を GAS に反映 |
| `clasp pull` | GAS の最新状態をローカルに取得 |

---

## よくあるトラブル

**Q. `clasp push` でエラーが出る**  
→ `clasp login` を再実行してください。トークンが切れている可能性があります。

**Q. `.clasp.json` を誤ってコミットしてしまった**  
→ 直ちにチームリーダーに報告し、GAS プロジェクトのアクセス権を見直してください。  
　 `.gitignore` に追加しただけでは既存の追跡は解除されません。以下も実行してください。

```bash
git rm --cached GASscripts/.clasp.json
git commit -m "fix: .clasp.json をトラッキングから除外"
```

**Q. `npm install -g @google/clasp` が Permission エラーになる（macOS）**  
→ 先頭に `sudo` を付けて実行してください。

```bash
sudo npm install -g @google/clasp
```
