# 要求仕様書：Nature Remo を用いたエアコン自動制御システム

---

## 1. 全体概要

### 1.1 システムの概要

Nature Remo 3 の内蔵温度センサから室内温度を定期的に取得し、設定したしきい値と比較することで、エアコンを自動的に ON / OFF 制御するシステム。  
ユーザはスプレッドシート上で室内環境の記録を確認できるとともに、LINE Notify によりエアコンの稼働状況をリアルタイムで通知として受け取ることができる。

### 1.2 製品の機能

5 分ごとに Nature Remo 3 のセンサから温度・湿度・照度を取得し、Google スプレッドシートに記録する。  
取得した温度データが設定したしきい値を超えた場合にエアコンを自動で ON にし、しきい値を下回った場合に自動で OFF にする。  
エアコンの稼働・停止のタイミングでは、LINE Notify を介してユーザに通知を送信する。  
さらに、1 時間ごとに室内の温度・湿度を定期通知する。

### 1.3 想定する利用者の特性

エアコンを利用するすべての人を対象とするが、特に、室内の温度管理を自ら行うことを煩わしく感じる人や、明確な指標に基づいてエアコンの設定を行いたい人に有益なシステムである。  
また、エアコン操作が難しい子供や高齢者にとっても活用できる。たとえば、子供が一人で在宅している状況において、室内温度が上昇しても子供がエアコンを操作できず熱中症に陥るリスクを軽減することが期待される。

## 2. システム構成

### 2.1 ハードウェア構成

| 機器名 | 型番・種別 | 役割 |
|--------|-----------|------|
| スマートリモコン | Nature Remo 3 | 温度・湿度・照度センサ、赤外線送信 |
| エアコン（家電） | 赤外線リモコン対応機器 | 制御対象 |
| 実行環境 | PC / Raspberry Pi | Python スクリプトの実行 |

### 2.2 ソフトウェア・サービス構成

| サービス／ライブラリ | 用途 |
|---------------------|------|
| Nature Remo Cloud API | センサデータ取得・赤外線信号送信 |
| Google Sheets API | センサデータの記録・しきい値設定の読み取り |
| LINE Notify API | エアコン稼働状況・室内環境の通知送信 |
| Python 3.x | 制御スクリプトの実行言語 |
| schedule ライブラリ | 定期実行処理の管理 |

---

## 3. 詳細

### 3.1 機能要求

- ユーザは、5 分ごとに記録される室内の温度・湿度・照度をスプレッドシート上で確認できること
- 温度がしきい値を超えた場合にエアコンが自動で稼働し、ユーザは LINE Notify により稼働日時・室内温度・湿度・設定温度を確認できること
- 温度がしきい値を下回った場合にエアコンが自動で停止し、ユーザは LINE Notify により停止日時・室内温度・湿度を確認できること
- ユーザは、エアコンの稼働状況に関わらず、1 時間ごとに室内の温度・湿度を LINE Notify で確認できること
- ユーザは、エアコンを稼働させる温度しきい値をスプレッドシートの所定のセルに入力することで設定できること
- ユーザは、エアコンの設定温度をスプレッドシートの所定のセルに入力することで設定できること

### 3.2 非機能要求

- エアコン稼働時・停止時の LINE 通知は、検知から 3 分以内に送信され、ユーザが確認可能であること
- センサデータの取得・記録処理は 5 分間隔で継続的に動作し、スクリプト起動後に人手による介入を必要としないこと
- しきい値設定の変更はスクリプト再起動なしに、次回の取得タイミングで反映されること

## 4. 処理フロー

| ステップ | 処理内容 | 実行間隔 |
|---------|---------|---------|
| ① データ取得 | Nature Remo API から温度・湿度・照度を取得 | 5 分ごと |
| ② データ記録 | 取得したセンサ値を Google スプレッドシートへ書き込み | 5 分ごと |
| ③ しきい値判定 | スプレッドシートのしきい値セルを読み取り、温度と比較 | 5 分ごと |
| ④ エアコン制御 | しきい値超過 → ON 命令 / 下回り → OFF 命令を赤外線送信 | 条件合致時 |
| ⑤ LINE 通知（制御時） | 稼働・停止時の室内環境と設定情報を LINE Notify で送信 | 制御発生時 |
| ⑥ LINE 通知（定期） | 室内温度・湿度を LINE Notify で定期報告 | 1 時間ごと |

---

## 5. 設定パラメータ一覧

| パラメータ名 | 設定箇所 | 初期値 | 説明 |
|------------|---------|-------|------|
| 冷房 ON 温度しきい値 | スプレッドシート B2 | 28.0 ℃ | この温度を超えると冷房を ON にする |
| 冷房 OFF 温度しきい値 | スプレッドシート B3 | 26.0 ℃ | この温度を下回ると冷房を OFF にする |
| エアコン設定温度 | スプレッドシート B4 | 26 ℃ | 冷房稼働時の設定温度 |
| データ取得間隔 | スクリプト定数 | 5 分 | センサ取得・記録の実行間隔 |
| 定期通知間隔 | スクリプト定数 | 60 分 | LINE 定期通知の送信間隔 |

## 6. 使用 API・認証情報

| API | 取得方法 | 使用用途 |
|-----|---------|---------|
| Nature Remo API トークン | Nature Remo アプリ → アカウント設定 | センサ取得・赤外線送信 |
| Google Service Account JSON | Google Cloud Console | スプレッドシートへの読み書き |
| LINE Notify トークン | LINE Notify 公式サイト | 通知メッセージの送信 |

---

## 7. 制約・前提条件

- エアコンは Nature Remo 3 に赤外線信号を事前に学習させておくこと
- 実行環境はインターネットに常時接続されていること
- Python 3.x および必要なライブラリ（`remo`, `gspread`, `schedule`, `requests` 等）がインストール済みであること
- スプレッドシートには Google Service Account が共有されていること
- 本システムは冷房のみを制御対象とし、暖房・除湿等のモード切替は対象外とする
