## 実装完了した機能

### 1. 高温アラート通知機能

**ファイル**: `GAS_scripts/discord.js`

**新規関数**: `sendHighTempAlertNotification(webhookUrl, temp, humidity, di)`

- **機能**: 室温が設定値（デフォルト 35℃）を超えた場合に Discord に自動通知
- **通知内容**:
  - タイトル: 「【高温アラート】」
  - 現在の室温・湿度・不快指数
  - 警告マーク付きメッセージ
- **対応要件**: 要求仕様書_コア機能実装.md
  - 「室温が設定した上限（デフォルト：35℃）を超えた場合、Discordの指定チャンネルへ「室内高温アラート」が送信されること」

### 2. 外出時機器リスト通知機能（準備完了）

**ファイル**: `GAS_scripts/discord.js`

**新規関数**: `sendOutdoorNotification(webhookUrl, deviceList, spreadsheetUrl)`

- **機能**: 外出時のオン状態機器リストを Discord に通知（Phase 2 対応準備）
- **通知内容**:
  - タイトル: 「【消し忘れ通知】」
  - オン状態の機器名・稼働時間・設定値
  - スプレッドシート操作ダッシュボードへのリンク

### 3. Discord 送信時のリトライ機能強化

**ファイル**: `GAS_scripts/discord.js`

**改善関数**: `sendDiscordMessage_(webhookUrl, content)`

- **強化内容**: 
  - リトライ機能の追加（最大 2 回の試行）
  - HTTP ステータスコード検証（200-299 を成功と判定）
  - 失敗時の詳細ログ出力
- **対応要件**: 要求仕様書_コア機能実装.md
  - 「Nature Remo API または Discord Webhook の呼び出しが失敗した場合、1 回リトライを実行し、それでも失敗した場合は Discord の指定チャンネルへエラー通知を送信すること」

### 4. スプレッドシート設定値の追加

**ファイル**: `GAS_scripts/sheet.js`

**変更内容**:
- **読み込み範囲**: `B2:B10` → `B2:B11` に拡張
- **新規設定値**: `highTempAlertThreshold`（高温アラート上限温度）
  - デフォルト値: 35℃
  - ユーザーは Settings シートの B11 セルで変更可能

### 5. 高温判定ロジックの統合

**ファイル**: `GAS_scripts/main.js`

**変更内容**:
- `runEvery5Minutes()` 内で高温アラート判定を追加
- **判定条件**: `sensorData.temp > settings.highTempAlertThreshold`
- **実行タイミング**: 5 分ごとのセンサ取得時に自動判定・通知
